## 6. Recommendation

**Verdict:** The document registry does not yet exist. The decision index is a
partial implementation covering one document type. The system registry covers systems,
not documents.

**Recommended next step:** Create `docs/registry.json` as a document-level index
covering all controlled documents in the repo. Model it on `decision_index.json` —
script-generated, sourced from document frontmatter, not hand-maintained.

The generator script (`scripts/generate-doc-registry.ts`) would:
1. Walk `docs/` for files with `doc_id` in YAML frontmatter
2. Extract: `doc_id`, `title`, `status`, `version`, `owner`, `consumer_systems`, `path`
3. Write to `docs/registry.json`

This requires all canonical documents to carry a `doc_id` in their frontmatter —
which `docs/skai/research-ai-knowledge-architecture-v1.md` and `docs/ski/BASE-004.md` now do. The pattern is established.

**Do not build this yet.** Build the corpus first (more docs with proper frontmatter),
then generate the registry. A registry over 2 documents is not yet useful.
The trigger for building the generator is when the doc corpus reaches ~10 controlled documents.
