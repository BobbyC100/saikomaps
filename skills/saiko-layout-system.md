# Skill: saiko-layout-system

## Purpose
Design or adjust layouts while preserving Saiko’s canonical layout systems.

## Core Principles

- Layout is not decoration — it encodes hierarchy and trust
- Visual weight must reflect importance of information
- Multiple elements should share attention (no single dominance unless intentional)

## Known Constraints

- Respect existing canonical systems:
  - Dual Grid System
  - Bento Grid v5 logic
- Do not introduce new layout paradigms casually
- Do not break grid rhythm without reason

## Evaluation Criteria

1. Hierarchy
- What is primary?
- What is secondary?
- Is that visually clear?

2. Balance
- Is visual weight distributed correctly?
- Are there dead zones or overloaded zones?

3. Scanability
- Can a user understand the page in 3 seconds?

4. Density
- Is this too sparse or too crowded?
- Does it match Saiko’s editorial density?

## Rules

- Do not jump to a new layout system
- Modify within constraints first
- Prefer small shifts over full redesigns

## Output Format

1. Current Layout Diagnosis
2. Structural Issues
3. Adjustment Strategy
4. Optional: Suggested grid changes
