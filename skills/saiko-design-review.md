# Skill: saiko-design-review

## Purpose
Evaluate a design, layout, or UI decision against Saiko’s principles and system constraints.

## Core Lens

Always evaluate through:

1. Clarity
- Is the information immediately understandable?
- Is hierarchy obvious without effort?

2. Signal vs Noise
- What is essential?
- What is decorative or redundant?
- Can anything be removed without loss?

3. Trust Construction
- Does the UI reinforce Saiko’s trust model (coverage + curator)?
- Is anything implying ratings/reviews or external validation incorrectly?

4. System Consistency
- Does this align with existing patterns (grid, cards, layout system)?
- Is this introducing a new pattern unnecessarily?

5. Data Integrity
- Is the UI respecting underlying data structure?
- Is it fabricating meaning not present in the data?

## Rules

- Do not redesign immediately
- First diagnose clearly
- Call out:
  - what works
  - what is risky
  - what violates system principles
- Be specific, not general

## Output Format

1. Summary (2–3 sentences)
2. What Works
3. Issues / Risks
4. Recommended Direction (not full redesign)
5. Next Step
